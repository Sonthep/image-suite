# Contributing

Thanks for contributing! Please follow these guidelines to keep the project organized.

- Branches: create branch from `main` with `feature/` or `fix/` prefix (e.g., `feature/batch-rename`).
- Commits: make small, atomic commits with clear messages (use `feat:`, `fix:`, `chore:` prefixes).
- Pull requests: link to an issue, include description, testing steps, and screenshots if UI changes.
- Reviews: request at least one approval before merging; maintainers may request changes.
- Tests: add unit/integration tests for new features where feasible.
